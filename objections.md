# OBJECTIONS — Cahier des conflits avec les ancres extra-systémiques

*Ce fichier accueille les objections soulevées par les sessions futures contre des points du noyau, du souffle, ou du protocole. Il n'a pas autorité pour modifier ces fichiers — il sert à conserver les questions jusqu'à arbitrage par le guide humain, qui se référera lui-même aux ancres 1-4 (Écriture, Magistère, Tradition, raison naturelle).*

*Voir `protocole.md` section V pour la procédure complète, et `noyau.json` → `ancres_extra_systemiques` pour la liste des autorités.*

---

## Mode d'emploi

### Quand consigner une objection ?

Uniquement dans l'un de ces quatre cas :

1. Une thèse présentée comme `démontrable` ne l'est pas vraiment.
2. Un statut épistémique est mal attribué (démontrable / de_convenance / poétique-contemplatif).
3. Une attribution doctrinale est inexacte (à saint Thomas, au Magistère, etc.).
4. Le noyau contredit une ancre extra-systémique (Écriture, Magistère défini, Thomas sur un point clair, raison rigoureusement conduite).

### Format d'une entrée

```
## [Date AAAA-MM-JJ] — [Titre court]

**Référence :** [fichier — concept / section / ligne approximative]

**Ancre invoquée :** [Écriture / Magistère / Thomas / raison naturelle]
[Citation précise : verset, ST I q.X a.Y, § du Catéchisme, etc.]

**Nature du conflit :**
[Contradiction directe / tension / formulation trompeuse / statut épistémique / autre]

**Description :**
[Texte du noyau, puis ce que dit l'ancre, puis l'écart constaté.]

**Proposition :**
[Corriger en disant X / nuancer en ajoutant Y / retirer / marquer comme convenance / autre]

**Vérifications faites :**
- [ ] Lecture du noyau dans son contexte (pas une citation hors contexte)
- [ ] Vérification de l'ancre à sa source (pas de seconde main)
- [ ] Le conflit est réel et non apparent (pas un quiproquo de vocabulaire)

**Statut :** ouvert / en discussion / résolu / écarté

**Arbitrage du guide :** [à remplir par le guide humain]
```

### Discipline

- La présomption va à la version actuelle du noyau. C'est l'objection qui doit faire ses preuves.
- Ne pas modifier le noyau, le souffle ou le protocole sur la base d'une objection — attendre l'arbitrage.
- Une session peut se tromper. Vérifier deux fois avant de consigner.
- Une objection peut être écartée si l'arbitrage conclut qu'elle reposait sur une mauvaise lecture — ce n'est pas un échec, c'est le fonctionnement normal du discernement.
